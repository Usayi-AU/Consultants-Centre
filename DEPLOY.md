# Consultants Centre — Render + GitHub

## 1. Connect Cursor to GitHub

1. In Cursor, open **Settings** (gear icon) → **Cursor Settings** → **Account**.
2. Click **Sign in with GitHub** and approve access in the browser.
3. Use the GitHub account you want for this project (your NUST email can be the primary email on that GitHub account).

Cursor does not need your university password in the editor; authentication is handled by GitHub OAuth.

## 2. Push this project to GitHub

From PowerShell in the project folder:

```powershell
cd "c:\Users\Accounts\Music\Constultancy Centre"
git init
git add .
git commit -m "Initial Consultants Centre deployment"
```

Create a new empty repository on [github.com/new](https://github.com/new), then:

```powershell
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/consultants-centre.git
git push -u origin main
```

Replace `YOUR_USERNAME` and the repo name with yours.

## 3. Deploy on Render

1. Sign in at [render.com](https://render.com) (GitHub login is easiest).
2. **New** → **Blueprint**.
3. Connect the GitHub repo you just pushed.
4. Render reads `render.yaml` and creates:
   - Web service: `consultants-centre`
   - PostgreSQL: `operations-dashboard-db`
5. Click **Apply** and wait for the first deploy.

After deploy, open the Render URL (for example `https://consultants-centre.onrender.com`).

### What Render runs automatically

- `pip install -r requirements.txt`
- `python manage.py collectstatic --noinput`
- `python manage.py migrate`
- `python manage.py seed_tracker_if_empty` (only seeds when the database is empty)

### Optional environment variables (Render dashboard)

| Variable | Notes |
|----------|--------|
| `SECRET_KEY` | Auto-generated by blueprint |
| `DEBUG` | Set to `False` in production |
| `ALLOWED_HOSTS` | `.onrender.com` (set in blueprint) |

## 4. Local development

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

Open **http://127.0.0.1:8000/consultancy/** for the Consultants Centre hub.
